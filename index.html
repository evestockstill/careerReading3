What is the mock function?

We replace something we do not control with something we do. It allows us features to
  • Capture calls
  • Set return values
  • Change the implementation

What is a callback?

It is a function passed to another function, when the first function is done the second will run. Problems arise when you begin nesting functions within functions the code is hard to read and may get out of order.

How can Promises help? What do they do?

Using arrow functions we can use the .then() method to un-nest the endless callbacks going in order. It notifies an asynchronous operation when it is done. You can use a catch() for an error if something goes wrong

Async/Await

Async declares the function will handle the asynchronous operations and the await is used to specify what we want to await. In order to be a await-able function it must have the async keyword or returns a promise

How is the code stack operating?

The JS runtime can only run one line of code at a time. It runs on a single thread operation. The first call in is the last call out. Without any modifications we have to wait for each line of code to finish before moving on to the next.
